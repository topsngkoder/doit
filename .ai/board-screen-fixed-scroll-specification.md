# Спецификация: экран доски с фиксированным фоном и фиксированными верхними панелями

## 1. Назначение документа
Документ фиксирует однозначные требования к экрану доски в приложении `Doit`.

Цель доработки:
- зафиксировать фон доски относительно viewport;
- исключить прокрутку верхней полосы приложения;
- исключить прокрутку полосы с названием доски;
- сделать область колонок единственной прокручиваемой зоной экрана доски;
- зафиксировать точные правила вертикальной и горизонтальной прокрутки без двусмысленности.

В этой спецификации нет кода. Все решения ниже обязательны.

## 2. Объект доработки
Под экраном доски в этом документе понимается страница `/boards/{boardId}` целиком, включая:
- верхнюю полосу приложения;
- полосу с названием доски;
- панель действий над колонками;
- рабочую область с колонками;
- фон доски.

## 3. Термины

### 3.1. Верхняя полоса приложения
Верхняя полоса приложения — это глобальный header приложения с навигацией.

### 3.2. Полоса доски
Полоса доски — это строка под глобальным header, в которой отображаются:
- название доски;
- участники;
- действия доски.

### 3.3. Панель действий над колонками
Панель действий над колонками — это строка под полосой доски, в которой находятся:
- кнопка создания колонки;
- служебные статусы и сообщения, относящиеся к области колонок.

Если в конкретном состоянии страницы в этой панели нет ни одного видимого элемента, панель не занимает высоту и считается отсутствующей.

### 3.4. Viewport колонок
Viewport колонок — это единственная область страницы, внутри которой разрешена прокрутка.

### 3.5. Фон доски
Фон доски — это визуальный фон страницы доски:
- изображение доски, если оно выбрано;
- базовый фон страницы, если изображение доски не выбрано.

## 4. Фиксированные решения

### 4.1. Общая модель экрана
Экран доски должен занимать всю высоту viewport браузера.

Экран доски делится на 4 вертикальные зоны сверху вниз:
1. верхняя полоса приложения;
2. полоса доски;
3. панель действий над колонками;
4. viewport колонок.

Зоны 1, 2 и 3 являются фиксированными.

Зона 4 является единственной прокручиваемой.

### 4.2. Какие зоны не прокручиваются
Не прокручиваются ни по вертикали, ни по горизонтали:
- верхняя полоса приложения;
- полоса доски;
- панель действий над колонками;
- фон доски.

Эти зоны должны оставаться визуально закреплёнными относительно viewport, пока пользователь находится на странице доски.

### 4.3. Единственный scroll-container
На странице доски должен существовать ровно один основной scroll-container: viewport колонок.

Только viewport колонок имеет право создавать:
- вертикальную прокрутку;
- горизонтальную прокрутку.

Недопустимы дополнительные конкурирующие scroll-container:
- на уровне `body`;
- на уровне корневого контейнера страницы доски;
- на уровне фона доски;
- на уровне верхней полосы приложения;
- на уровне полосы доски;
- на уровне панели действий над колонками;
- на уровне отдельной колонки.

### 4.4. Правило для вертикальной прокрутки
Если полная высота содержимого колонок больше доступной высоты viewport колонок, вертикальная прокрутка возникает только внутри viewport колонок.

При вертикальной прокрутке:
- верхняя полоса приложения остаётся на месте;
- полоса доски остаётся на месте;
- панель действий над колонками остаётся на месте;
- фон доски остаётся на месте;
- перемещается только содержимое области колонок.

### 4.5. Правило для горизонтальной прокрутки
Если суммарная ширина всех колонок, межколоночных отступов и внутренних отступов рабочей области больше доступной ширины viewport колонок, горизонтальная прокрутка возникает только внутри viewport колонок.

При горизонтальной прокрутке:
- верхняя полоса приложения не смещается;
- полоса доски не смещается;
- панель действий над колонками не смещается;
- фон доски не смещается;
- перемещается только горизонтальная лента колонок.

### 4.6. Правило для колонок
Колонки остаются частью единого горизонтального ряда.

В рамках этой задачи:
- не вводится режим, в котором колонки перестраиваются в один вертикальный столбец;
- не вводится режим, в котором каждая колонка получает собственную внутреннюю вертикальную прокрутку;
- не вводится режим, в котором верхние панели начинают скроллиться вместе с колонками.

На любых ширинах экрана, начиная с минимально поддерживаемой ширины интерфейса, используется одна и та же модель:
- верхние панели фиксированы;
- горизонтальный ряд колонок находится в viewport колонок;
- при нехватке места используется горизонтальная прокрутка внутри viewport колонок;
- при нехватке высоты используется вертикальная прокрутка внутри viewport колонок.

### 4.7. Правило для высоты экрана
Высота viewport колонок вычисляется по формуле:

`высота viewport браузера минус фактическая высота верхней полосы приложения минус фактическая высота полосы доски минус фактическая высота панели действий над колонками`.

Иной источник вертикальной прокрутки на странице доски не допускается.

Если высота любой из фиксированных верхних зон меняется из-за:
- изменения ширины окна;
- переноса элементов на новую строку;
- появления или исчезновения служебного сообщения;
- изменения масштаба интерфейса;

высота viewport колонок должна быть немедленно пересчитана так, чтобы общая страница по-прежнему помещалась в viewport без появления прокрутки у `body`.

### 4.8. Правило для фона доски
Фон доски должен быть закреплён относительно viewport, а не относительно прокручиваемого содержимого колонок.

Это означает:
- при вертикальной прокрутке колонок фон доски не движется;
- при горизонтальной прокрутке колонок фон доски не движется;
- фон доски остаётся визуально на одном и том же месте до тех пор, пока пользователь не меняет сам фон или не покидает страницу.

Если фон доски является изображением, применяются обязательные правила:
- изображение заполняет доступную область экрана доски без пустых полос;
- изображение не повторяется;
- изображение остаётся центрированным;
- прокрутка колонок не создаёт эффекта параллакса;
- прокрутка колонок не меняет точку привязки изображения.

Если изображение фона отсутствует, базовый фон страницы доски также считается закреплённым и не участвует в прокрутке.

### 4.9. Правило для фиксированных верхних зон
Верхние зоны образуют один непрерывный фиксированный каркас страницы доски.

Порядок сверху вниз обязателен:
1. верхняя полоса приложения;
2. полоса доски;
3. панель действий над колонками;
4. viewport колонок.

Менять этот порядок в рамках задачи нельзя.

### 4.10. Правило для ширины фиксированных зон
Верхняя полоса приложения, полоса доски и панель действий над колонками всегда привязаны к ширине viewport.

Горизонтальная прокрутка колонок не должна:
- сдвигать эти зоны влево или вправо;
- менять их ширину;
- заставлять их участвовать в горизонтальном переполнении.

### 4.11. Поведение при малом объёме данных
Если все колонки полностью помещаются в доступную ширину и высоту viewport колонок:
- горизонтальная прокрутка отсутствует;
- вертикальная прокрутка отсутствует;
- верхние панели остаются фиксированными;
- фон доски остаётся фиксированным.

### 4.12. Поведение при большом объёме данных
Если колонки не помещаются по ширине, по высоте или одновременно по обеим осям:
- обе прокрутки, если они нужны, появляются только у viewport колонок;
- страница в целом не должна начинать прокручиваться;
- пользователь не должен терять из вида верхнюю полосу приложения;
- пользователь не должен терять из вида полосу доски;
- пользователь не должен терять из вида панель действий над колонками.

### 4.13. Что не меняется этой задачей
Эта задача не меняет:
- состав элементов в верхней полосе приложения;
- состав элементов в полосе доски;
- состав элементов в панели действий над колонками;
- бизнес-логику доски;
- права доступа;
- тексты;
- размеры и порядок самих колонок;
- визуальный стиль карточек и колонок, кроме изменений, которые необходимы для соблюдения правил прокрутки и фиксации.

## 5. Недопустимое поведение
Ошибкой считаются следующие состояния:
- прокрутка страницы происходит у `body` или у корневого контейнера страницы доски;
- фон доски уезжает вместе с колонками;
- верхняя полоса приложения уезжает вверх при вертикальной прокрутке колонок;
- полоса доски уезжает вверх при вертикальной прокрутке колонок;
- панель действий над колонками уезжает вверх при вертикальной прокрутке колонок;
- верхние панели смещаются влево или вправо при горизонтальной прокрутке колонок;
- внутри отдельной колонки появляется собственный вертикальный scroll-container;
- на странице возникает более одного основного scroll-container;
- при изменении высоты верхних панелей у страницы появляется внешний вертикальный скролл;
- на узком экране колонки перестраиваются в один столбец вместо горизонтальной ленты с прокруткой.

## 6. Критерии приёмки
Задача считается выполненной, если одновременно соблюдаются все условия:

1. На странице `/boards/{boardId}` фон доски визуально закреплён и не двигается ни при вертикальной, ни при горизонтальной прокрутке колонок.
2. Верхняя полоса приложения всегда остаётся видимой и неподвижной относительно viewport.
3. Полоса доски с названием всегда остаётся видимой и неподвижной относительно viewport.
4. Панель действий над колонками не прокручивается вместе с колонками.
5. Единственная прокручиваемая область страницы доски — viewport колонок.
6. Если колонки не помещаются по ширине, появляется только горизонтальная прокрутка viewport колонок.
7. Если колонки не помещаются по высоте, появляется только вертикальная прокрутка viewport колонок.
8. Если колонки не помещаются одновременно по ширине и высоте, обе прокрутки работают только внутри viewport колонок.
9. При прокрутке колонок по любой оси `body` не получает собственный scroll.
10. На узких экранах сохраняется горизонтальная модель доски; колонки не складываются в один столбец.

## 7. Статус вопросов
Открытых вопросов для этой спецификации нет.
