# Спецификация: экран доски с фиксированным full-screen фоном

## 1. Назначение документа
Документ фиксирует окончательные требования к экрану доски в приложении Doit.

Цель доработки:
- фон доски должен быть фиксированным;
- фон доски должен покрывать всю область доски по ширине и высоте;
- область доски не должна быть визуально центрирована внутри узкого контейнера;
- нижний лишний scroll страницы должен быть устранён;
- название доски, участники, настройки доски и панель действий над колонками должны оставаться над фиксированным фоном;
- единственной прокручиваемой областью должен быть viewport колонок.

Этот документ является приоритетным для задачи про фон и scroll экрана доски. Все решения ниже обязательны.

## 2. Объект доработки
Под экраном доски в этом документе понимается маршрут просмотра конкретной доски целиком.

Экран доски состоит из двух частей:
1. глобальный header приложения;
2. область доски под глобальным header.

В область доски входят:
- полоса доски с названием;
- блок участников;
- меню настроек доски;
- панель действий над колонками;
- рабочая область колонок;
- фон доски.

Фон этой задачи не распространяется на глобальный header приложения. Глобальный header остаётся отдельной верхней зоной приложения.

## 3. Обязательная геометрия экрана

### 3.1. Высота
После отрисовки глобального header область доски обязана занимать всю оставшуюся высоту viewport до нижней границы окна браузера.

Нижняя граница области доски обязана совпадать с нижней границей viewport.

Свободной полосы под областью доски быть не должно.

### 3.2. Ширина
Область доски обязана занимать всю доступную ширину viewport.

Для маршрута доски запрещены:
- центрирование области доски;
- ограничение максимальной ширины области доски;
- визуальный режим, при котором область доски выглядит как узкая колонка по центру экрана.

Левая граница области доски обязана совпадать с левой границей viewport.
Правая граница области доски обязана совпадать с правой границей viewport.

### 3.3. Фоновая область
Фон доски обязан покрывать всю область доски целиком:
- от верхней границы полосы доски;
- до нижней границы viewport;
- от левой границы viewport;
- до правой границы viewport.

Фон обязан находиться под полосой доски, под блоком участников и настроек, под панелью действий над колонками и под областью колонок.

## 4. Вертикальная структура области доски
Область доски всегда состоит из трёх зон сверху вниз:

1. полоса доски;
2. панель действий над колонками;
3. viewport колонок.

Полоса доски всегда существует.

Панель действий над колонками существует только если в ней есть хотя бы один видимый элемент. Если видимых элементов нет, эта зона не занимает высоту вообще.

Viewport колонок всегда занимает весь остаток высоты после вычитания фактической высоты верхних зон области доски.

## 5. Полоса доски
Полоса доски — это верхняя зона области доски, в которой расположены:
- название доски;
- участники;
- настройки доски.

Обязательные правила:
- полоса доски не участвует ни в вертикальной, ни в горизонтальной прокрутке;
- полоса доски всегда видима, пока пользователь находится на маршруте доски;
- полоса доски находится поверх фиксированного фона доски;
- горизонтальная прокрутка колонок не сдвигает полосу доски влево и вправо;
- вертикальная прокрутка колонок не уводит полосу доски вверх.

## 6. Панель действий над колонками
Панель действий над колонками — это зона между полосой доски и viewport колонок.

В этой зоне могут находиться:
- кнопка создания колонки;
- служебные статусы;
- сообщения, относящиеся к работе области колонок.

Обязательные правила:
- панель не участвует ни в вертикальной, ни в горизонтальной прокрутке;
- панель находится поверх фиксированного фона доски;
- если панель отображается, она всегда полностью видима;
- если панель скрыта, её высота равна нулю;
- появление или исчезновение панели не должно вызывать scroll страницы.

## 7. Viewport колонок
Viewport колонок — это единственная область страницы доски, внутри которой разрешена прокрутка.

Только viewport колонок имеет право создавать:
- вертикальную прокрутку;
- горизонтальную прокрутку.

Viewport колонок обязан:
- занимать весь остаток высоты области доски;
- занимать всю ширину области доски;
- быть расположен под фиксированными верхними зонами;
- не создавать дополнительный внешний отступ, из-за которого снизу появляется пустая полоса или лишний scroll.

## 8. Правила прокрутки

### 8.1. Единственный основной scroll-container
На странице доски должен существовать ровно один основной scroll-container.

Этим контейнером является только viewport колонок.

Недопустимы основные scroll-container:
- у документа;
- у body;
- у корневого контейнера приложения;
- у корневого контейнера маршрута доски;
- у фонового слоя доски;
- у полосы доски;
- у панели действий над колонками;
- у отдельной колонки.

### 8.2. Вертикальная прокрутка
Если содержимое колонок не помещается по высоте, вертикальная прокрутка появляется только внутри viewport колонок.

При вертикальной прокрутке:
- движется только содержимое области колонок;
- фон доски не движется;
- полоса доски не движется;
- панель действий над колонками не движется;
- глобальный header приложения не движется.

### 8.3. Горизонтальная прокрутка
Если суммарная ширина ленты колонок не помещается по ширине, горизонтальная прокрутка появляется только внутри viewport колонок.

При горизонтальной прокрутке:
- движется только горизонтальная лента колонок;
- фон доски не движется;
- полоса доски не движется;
- панель действий над колонками не движется;
- глобальный header приложения не движется.

### 8.4. Лишний нижний scroll
Нижний лишний scroll считается ошибкой.

Под лишним нижним scroll в этой задаче понимается любое состояние, при котором:
- у страницы появляется вертикальная прокрутка из-за области доски;
- под фиксированным фоном доски видна пустая зона;
- нижняя граница области доски не совпадает с нижней границей viewport;
- scroll возникает из-за центрирующего контейнера, внешних паддингов, внешних margin, лишнего min-height или конкурирующего overflow-контейнера.

После доработки ни один из этих сценариев не допускается.

## 9. Правила фона доски

### 9.1. Фиксация
Фон доски должен быть закреплён относительно области доски и визуально оставаться неподвижным при любой прокрутке колонок.

Это означает:
- при вертикальной прокрутке фон не смещается;
- при горизонтальной прокрутке фон не смещается;
- прокрутка колонок не создаёт параллакс;
- точка привязки фонового изображения не меняется.

### 9.2. Покрытие
Фон обязан непрерывно покрывать всю область доски без пустых полос:
- сверху под полосой доски;
- снизу до нижней границы viewport;
- слева до левой границы viewport;
- справа до правой границы viewport.

### 9.3. Правила для изображения
Если у доски выбран фон-изображение, применяются обязательные решения:
- изображение заполняет всю фоновую область доски;
- изображение не повторяется;
- изображение центрировано;
- изображение не обрезается так, чтобы появлялись пустые полосы по краям;
- прокрутка колонок не влияет на положение изображения.

### 9.4. Правила для режима без изображения
Если у доски нет фонового изображения, вместо него используется сплошной базовый фон доски.

Этот базовый фон подчиняется тем же правилам:
- он покрывает всю область доски;
- он не движется при прокрутке колонок;
- он не заканчивается раньше нижней или боковой границы области доски.

## 10. Правила для колонок
В рамках этой задачи колонки остаются частью одной горизонтальной ленты.

Обязательные правила:
- колонки не перестраиваются в один вертикальный столбец;
- горизонтальная прокрутка, если она нужна, живёт только в viewport колонок;
- отдельная колонка не получает собственного постоянного вертикального scroll-container;
- верхние зоны страницы доски не становятся частью ленты колонок.

## 11. Поведение при изменении размеров окна
Если меняется ширина окна, высота окна или масштаб интерфейса, экран доски обязан сразу сохранять ту же модель:
- область доски по-прежнему занимает всю ширину viewport;
- область доски по-прежнему занимает всю высоту под глобальным header;
- фон доски по-прежнему покрывает всю область доски;
- page-scroll не появляется;
- единственным scroll-container остаётся viewport колонок.

Если верхние зоны области доски становятся выше из-за переноса содержимого или появления сообщений, viewport колонок обязан уменьшиться ровно на эту разницу.

Увеличение высоты верхних зон не должно вызывать scroll страницы.

## 12. Что не меняется этой задачей
Эта задача не меняет:
- состав глобального header;
- состав полосы доски;
- состав участников;
- состав меню настроек доски;
- бизнес-логику доски;
- права доступа;
- тексты интерфейса;
- порядок колонок;
- визуальный стиль карточек и колонок, кроме изменений, необходимых для full-screen каркаса и правильной scroll-модели.

## 13. Недопустимое поведение
Ошибкой считаются следующие состояния:
- область доски ограничена по ширине и выглядит центрированной;
- фон доски не доходит до левого или правого края viewport;
- фон доски не доходит до нижней границы viewport;
- под фоном видна пустая полоса;
- у страницы есть лишний нижний scroll;
- у body есть scroll из-за доски;
- у корневого контейнера приложения есть scroll из-за доски;
- полоса доски прокручивается вместе с колонками;
- панель действий над колонками прокручивается вместе с колонками;
- фон доски движется вместе с колонками;
- верхние зоны смещаются влево или вправо при горизонтальной прокрутке колонок;
- на странице есть более одного основного scroll-container;
- отдельная колонка получает собственный постоянный вертикальный scroll-container;
- на узком экране колонки складываются в один столбец.

## 14. Критерии приёмки
Задача считается выполненной только если одновременно выполняются все условия:

1. Область доски под глобальным header занимает всю ширину viewport без центрирования и без ограничения максимальной ширины.
2. Область доски под глобальным header занимает всю оставшуюся высоту viewport до самого низа окна.
3. Фон доски покрывает всю область доски по ширине и высоте.
4. Фон доски визуально неподвижен при вертикальной и горизонтальной прокрутке колонок.
5. Название доски, участники и настройки доски остаются поверх фиксированного фона и не прокручиваются вместе с колонками.
6. Панель действий над колонками не прокручивается вместе с колонками.
7. Единственная прокручиваемая область страницы доски — viewport колонок.
8. Если места по ширине не хватает, горизонтальная прокрутка появляется только внутри viewport колонок.
9. Если места по высоте не хватает, вертикальная прокрутка появляется только внутри viewport колонок.
10. Если места не хватает одновременно по ширине и высоте, обе прокрутки работают только внутри viewport колонок.
11. У страницы отсутствует лишний нижний scroll.
12. Под фоном доски отсутствует пустая полоса.
13. Глобальный header, полоса доски и панель действий над колонками не смещаются при прокрутке колонок по любой оси.
14. На узких экранах колонки остаются горизонтальной лентой.

## 15. Статус вопросов
Открытых вопросов для этой спецификации нет.
